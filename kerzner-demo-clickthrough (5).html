05 Bridge · Now Identity

Identity Resolution

Data is clean. Now the platform decides which records belong to the same person — and which don't.

  • 4 match rules — fuzzy name + email, fuzzy name + phone, exact email, exact phone
  • Never name alone — Houssam and Hussam stay separate
  • Reconciliation — per-field source authority
  • Match clusters — auditable in Profile Explorer
Outcome: 21 source records → 12 unified individuals. Houssam unified across 4 records. Hussam held separate.
MATCH RULE EXAMPLE
IF fuzzy(name) AND exact(email)
  THEN merge_records()

IF fuzzy(name) AND exact(phone)
  THEN merge_records()

NEVER match(name) alone
